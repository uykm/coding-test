# coding-test

코딩 테스트 스터디 풀이 저장소. 월별 폴더(`2026-June`, ...)에 문제 풀이 문서를 모은다.

## 파일명 규칙

```
YYMMDD_알고리즘유형_플랫폼-번호_문제이름.md
```

- **YYMMDD**: 푼 날짜. 자동으로 오늘 날짜를 쓰지 말고 **사용자에게 물어본다**. 예: `260608`
- **알고리즘유형**: 문제 풀이에 쓰는 주요 알고리즘/자료구조 유형. 한글로 적는다.
  - 예: `투포인터`, `슬라이딩윈도우`, `그리디`, `해시`, `정렬`, `배열`, `문자열`, `프리픽스`, `수학`
- **플랫폼-번호**: 문제 출처와 번호
  - `LC-88` — LeetCode
  - `PG-42576` — 프로그래머스(Programmers)
- **문제이름**: 케밥-케이스. 영문 문제는 영어 슬러그, 한글 문제는 한글을 하이픈으로 연결
  - 예: `merge-sorted-array`, `완주하지-못한-선수`

예시:
- `260608_투포인터_LC-88_merge-sorted-array.md`
- `260608_해시_PG-42576_완주하지-못한-선수.md`

## 문제 링크로 파일 자동 생성

사용자가 문제 **링크만** 주면 아래 절차로 풀이 문서를 만든다.

1. **링크에서 정보 추출** (필요하면 WebFetch로 페이지를 읽어 제목/번호 확인)
   - LeetCode `leetcode.com/problems/<slug>` → 플랫폼 `LC`. URL에 번호가 없으므로 페이지를 fetch해 문제 번호와 제목을 가져온다. 파일 이름은 `<알고리즘유형>_LC-<번호>_<slug>`.
   - 프로그래머스 `programmers.co.kr/learn/courses/30/lessons/<번호>` → `PG-<번호>`. 제목은 페이지에서 가져와 한글 케밥-케이스로.
2. **날짜**: 사용자에게 날짜(YYMMDD)를 물어본다.
3. **알고리즘유형**: 문제의 주요 풀이 유형을 한글로 정한다(예: `투포인터`, `슬라이딩윈도우`). 애매하면 사용자에게 확인.
4. **폴더**: 별도 지정이 없으면 가장 최근 월 폴더(`2026-June`...)에 넣는다. 새 달을 시작하면 사용자에게 확인.
5. **내용**: `.template/problem.md`를 복사해 채운다.
   - `# 문제 제목` → 실제 문제 제목
   - `🔗 ` 뒤에 문제 링크
   - 풀이 섹션은 비워둔 채로 둔다(사용자가 직접 채움)
6. 파일은 생성만 하고 커밋은 사용자가 요청할 때만 한다.
